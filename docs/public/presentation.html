Code Repo Knowledge Layer — Problem

The Problem We Started With

Code repositories contain three kinds of knowledge:

  • Explicit knowledge — code, comments, README. Already in the repo, but hard to navigate.
  • Structural knowledge — "if you change this, that breaks." Implicit in code but impossible to trace manually at scale.
  • Contextual knowledge — "why was this written this way?" Lives only in people's heads. Lost completely when communication is absent.
Root Cause

Teams don't communicate around code — so new members must read everything manually, and it takes too long to understand what each part does.

Existing tools only cover

GitHub Copilot and search tools handle explicit knowledge. Structural and contextual knowledge still require asking people.

Code Repo Knowledge Layer — Brainstorm

Five Directions We Explored

1. AI Coaching Agent — "Repo Onboarding Buddy"

Ask "how does the data pipeline work?" and get an answer grounded in code structure, dependencies, and git history — not just docs.

2. Auto-Documentation

Living docs that auto-update when code changes. Four layers: one-liner, feature description, architecture context, business impact.

3. Usage Tracking Layer

Git history + AST analysis → Core Assets, Stable components, Dead Code candidates. Bus factor alerts. Tableau treemap visualization.

4. Interactive Code Map

Navigable graph of the entire repo. Click to drill down. Data lineage visualization. Cross-team dependency discovery.

5. Communication Gap Filler

Weekly AI newsletter of what changed. Cross-team impact alerts. "Ask the repo" Slack bot.

One-line product definition

"A VS Code Extension that lets you understand every context of a repo without leaving the editor."

Part 2

The Pivot

Why we shifted from Repo Buddy to Git Abstraction Tool

The Pivot — What We Hit

Barriers We Encountered

  • No shared DB or server — Ryan's guidance: "don't spend any money." SQLite local-only was the ceiling for Phase 1.
  • Cold start problem — Q&A bot needs senior engineers to answer first, but they already use Copilot for code explanation.
  • Weak value proposition — "What does this function do?" is already answered by Claude and Copilot. We weren't differentiated.
  • Tribal knowledge gap — The real value ("why was this written this way?") requires git history + PR discussion — hard to build reliably.
Ryan's Key Feedback (3/6 Honda Meeting)

"Think of this project as a new approach to version control. A new generation of GitHub or Git."

Ryan's Key Feedback (4/3 Honda Meeting)

"How do we make vibe coding easier? Ppl do not know what they want — AI can give a high-level plan. Think about the vibe coders using phones or tablets to code."

Git Abstraction Tool — Overview

What It Is

A cross-platform Electron desktop app that wraps git in a plain-language interface. No terminal. No git jargon. No learning curve.

Built local-first: full version control works completely offline. AI features are optional enhancements layered on top, never requirements.

Git Abstraction Tool — AI Features

6 Core AI Components

All optional · All privacy-preserving · All with graceful fallback

AI Feature 1 of 6

Auto Commit Message

When a user clicks Save Progress, the app reads the staged git diff and asks the AI to produce one plain-language sentence — no file paths, no git jargon.

"Updated the homepage background color and fixed typos in the contact form."

The generated message is pre-filled for review. The user edits if needed, then confirms. The internal summary is also stored for Weekly Report and Natural Language Undo to reference later.

  • Reads staged diff, file paths, and change types (new / modified / deleted)
  • 3–5 second timeout — saving is never blocked by AI
  • Per-project consent required before any diff is sent to AI
  • Auto-Save mode: draft → confirm → commit in one flow
  • Deterministic fallback message when AI is unavailable
  • Internal AI summary stored to power the other three AI features
AI Feature 2 of 6

Weekly Report

At the end of any selected week, the app compiles git stats and the stored AI summaries from each save into a readable narrative of what was accomplished.

Example Output

"This week you redesigned the portfolio homepage, fixed spacing issues on mobile, and added a new contact form. 14 saves across 3 active days — 847 lines added, 312 removed."

Useful for status updates, client progress logs, or personal portfolios — no git knowledge needed to generate or read it.

  • Combines git stats with stored AI commit summaries from the same week
  • Shows total saves, files changed, lines added/removed, active days
  • Cached by commit signature to avoid re-processing the same week
  • Stats-only fallback (no AI narrative) when AI is not connected
  • Distinguishes commits that have AI summaries vs. message-only entries
AI Feature 3 of 6

Natural Language Undo

Instead of scrolling through commit hashes, users describe in plain language when they want to return to.

"Go back to before I changed the upload flow."
"Restore to the state two days ago when the button was still red."

The AI searches up to 120 recent commits enriched with stored AI summaries, finds the best matching restore point, then shows a file-level preview before doing anything. The user must explicitly confirm.

  • Searches up to 120 recent commits, each enriched with their AI summary
  • Returns a primary match + 0–2 alternatives with confidence scores
  • Shows which files would be restored and which would be removed — before applying
  • User must explicitly confirm — nothing is applied automatically
  • Fallback options when the matched commit has no file-level delta
AI Feature 4 of 6

Smart Conflict Resolver

When a pull or merge creates git conflicts, the app automatically surfaces a guided dialog that walks the user through resolution — no raw <<<<<<< HEAD markers exposed.

Optional AI enhancement provides a plain-language explanation of which version to keep and why.

AI Hint Example

"Keep your version — it has the new layout changes. The incoming version only adds a minor color fix that's already in yours."

  • Dialog auto-opens when status.has_conflicts becomes true after pull/merge
  • "Keep Mine" / "Keep Theirs" per file — no diff syntax required
  • AI explains the tradeoff for each conflicted file on request
  • Once all files resolved, AI can suggest the merge commit message
  • Visual progress: resolved files show a checkmark badge
  • Abort button always visible to cancel the entire merge safely
AI Feature 5 of 6

File Insight

Click any tracked file in the file panel and the app instantly generates a plain-language explanation of what that file does, how it works, and which related files connect to it.

Example Output

"This is the main entry point of the React UI. It initializes the app, loads your saved preferences, and wires together the sidebar, file panel, and action panel. It coordinates the commit flow from file selection through to the save confirmation."

Explains any code file in plain language — no terminal or code knowledge required.

  • Scores related files via co-commit history, directory proximity, and name similarity — no hallucinated paths
  • 64 KB read limit; 12 KB content snippet sent to AI
  • Binary files auto-detected by extension and null-byte scan and skipped
  • Related file chips are clickable — analyze connected files in sequence
  • Requires AI connection; gracefully disabled when offline
  • Request deduplication prevents stale results on rapid file switching
AI Feature 6 of 6

Untracked File Review

When untracked files exist and AI is connected, a "Review untracked" button appears. The app classifies every file — should it be committed to the repo, or safely deleted?

Example Recommendations

Commit: src/Button.tsx (94%) — "Looks like new source code, likely belongs in version control."

Delete: node_modules/ (99%) — "Generated dependency folder, should never be committed."

Stage all commit-recommended files in one click, or select delete-recommended files for instant removal.

  • Two-tier decision — instant rule-based pass first, AI only for ambiguous cases; fast and low token cost
  • Rules auto-identify: build artifacts, caches, virtual envs, .env secrets, lock files, source directories
  • AI context: up to 100 files; up to 16 receive content snippets
  • Adaptive timeout: 12s base + 220ms per file, capped at 45s
  • Conservative fallback on timeout: unresolved files default to "commit"
  • Live UI updates as deletions complete; failed paths reported separately
