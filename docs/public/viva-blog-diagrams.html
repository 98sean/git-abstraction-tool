Diagram 3 — Product
Safety-First: Raw Git Terminal vs. VIVA
⚠️ Raw Git Terminal
✗ Silent git init in the wrong directory
✗ Force-push overwrites main with no warning
✗ Raw conflict markers (<<<< HEAD) exposed to user
✗ Auto pull can silently break working code
✗ Default branch deletion: no confirmation required
✗ No visibility into staged vs. untracked files
🛡️ With VIVA
✓ Repo init requires explicit folder selection by user
✓ Force-push blocked by default; user must consciously enable
✓ Guided file-by-file conflict dialog — no raw markers shown
✓ Pull Preview: inspect incoming changes before merging
✓ Branch deletion requires explicit two-step confirmation
✓ Full File Panel: tracked, staged, and untracked in one view
Core principle: Every destructive or irreversible action requires explicit, conscious user consent.
Diagram 4 — Product
The 6 Core AI Components
01
Auto Commit Message
Reads staged diff → generates a plain-language sentence. 3–5 s timeout so saving is never blocked.
02
Natural Language Undo
"Go back to before I changed the upload flow" — matches stored summaries, previews the diff before rollback.
03
Untracked File Review
Two-tier engine: rule-based pass handles node_modules / .env instantly; AI only for ambiguous files.
04
File Insight
Read-only explanation of a file's purpose and relationships, enriched with recent commit history.
05
Smart Conflict Resolver
Guided file-by-file dialog. AI explains the tradeoff and drafts the merge message once resolved.
06
Weekly Report
Git stats + stored AI summaries compiled into a readable narrative of everything accomplished that week.
All 6 components gracefully degrade when the AI API is unavailable — VIVA's core safety features never depend on AI.
